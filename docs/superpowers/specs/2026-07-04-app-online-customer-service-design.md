# APP 在线客服页设计

## 1. 背景与目标

当前 `prototype/batch-video-prototype.html` 原型中：

- 「信息咨询」页（`page-contact`）有「联系在线客服」按钮，但点击无响应。
- 「我的 → 联系客服」页（`page-service`）有「在线客服」按钮，同样无响应。

本设计为 APP 端补充一个**静态全屏在线客服聊天页**，让这两个入口可跳转，完善可交互原型的页面闭环。

## 2. 设计决策

- **页面形态**：独立全屏页面 `page-chat`，顶部返回按钮，底部输入栏。
- **交互深度**：静态原型。聊天内容、快捷问题、历史消息均为写死展示；输入框和发送按钮点击后 toast 提示「功能开发中，敬请期待」。
- **风格**：与现有 APP 原型保持一致（圆角气泡、蓝色主色、白色底部栏、隐藏底部导航）。

## 3. 页面结构

```
page-chat
├── 顶部导航栏（h-12，白色底，下边框）
│   ├── 返回按钮 ←
│   └── 页面标题：在线客服
├── 聊天内容区（flex-1，overflow-y-auto，灰色背景）
│   ├── 时间戳：今天 9:41
│   ├── 客服欢迎语气泡（左侧，白色）
│   ├── 快捷问题标签（左侧，白色小按钮）
│   ├── 用户消息气泡（右侧，蓝色）
│   └── 客服回复气泡（左侧，白色）
└── 底部输入栏（h-14，白色底，上边框）
    ├── 输入框 placeholder：请输入您想咨询的问题…
    └── 发送按钮
```

## 4. 页面元素与内容

### 4.1 顶部导航栏

| 元素 | 说明 |
|---|---|
| 返回按钮 | 左侧箭头图标，点击调用 `goBack()` |
| 页面标题 | 「在线客服」居中 |

### 4.2 聊天内容区

| 元素 | 位置 | 内容 |
|---|---|---|
| 时间戳 | 居中 | 今天 9:41（灰色小字） |
| 客服头像 | 左侧 | 蓝色圆形背景 + 白色客服图标 |
| 欢迎语 | 左侧气泡 | 「您好，欢迎来到批量剪辑！我是您的专属客服，请问有什么可以帮您？」 |
| 快捷问题 | 左侧标签行 | 「如何使用AI云创」「去水印支持哪些平台」「算力如何充值」 |
| 用户示例消息 | 右侧气泡 | 「如何使用AI云创？」 |
| 客服示例回复 | 左侧气泡 | 「进入 AI 云创 → 新增视频素材 → 上传或分割视频 → 配置分镜头 → 点击 AI 生成即可。」 |

### 4.3 底部输入栏

| 元素 | 说明 |
|---|---|
| 输入框 | 灰色圆角背景，placeholder「请输入您想咨询的问题…」 |
| 发送按钮 | 蓝色/灰色，点击 `showToast('功能开发中，敬请期待')` |

## 5. 入口与导航

| 入口页面 | 触发元素 | 跳转目标 |
|---|---|---|
| `page-contact`（信息咨询） | 「联系在线客服」按钮 | `page-chat` |
| `page-service`（联系客服） | 「在线客服」按钮 | `page-chat` |
| 右侧快捷导航面板 | 新增「在线客服」导航项 | `page-chat` |

返回时通过 `pageHistory` 回到入口页面。

## 6. 视觉规范

- 页面背景：灰色（`bg-gray-50`）。
- 客服气泡：白色背景，灰色细边框，圆角。
- 用户气泡：`bg-primary` 蓝色背景，白色文字，圆角。
- 头像：蓝色圆形（`bg-primary`），白色客服图标。
- 快捷问题标签：白色背景，灰色边框，蓝色文字，小圆角。
- 底部输入栏：白色背景，上边框灰色，与现有底部导航同高（`h-14`）。
- 该页面为二级页面，**不显示 APP 底部导航栏**。

## 7. 异常与边界

- 底部导航栏在聊天页隐藏：复用 `updateAppTabBar` 逻辑，因为 `page-chat` 不在四个主 tab 中。
- 返回路径正确：通过 `goBack()` 回到 `page-contact` 或 `page-service`。
- 静态输入无真实发送：点击输入框聚焦或发送按钮时，统一 toast 提示功能开发中。

## 8. 实现范围

本次仅修改 `prototype/batch-video-prototype.html`：

1. 新增 `page-chat` 页面 DOM。
2. 修改 `page-contact` 和 `page-service` 的「在线客服」按钮 `onclick`。
3. 在右侧 `navPanel` 增加「在线客服」入口。
4. 无需新增 CSS/JS 文件，样式复用现有 Tailwind 工具类。

## 9. 方案对比

| 方案 | 说明 | 结果 |
|---|---|---|
| 独立静态客服页 | 新建 `page-chat`，完整聊天界面 | **选定** |
| 聊天 + 服务聚合页 | 在 `page-service` 中嵌入聊天区 | 聊天区被压缩，不符合全屏对话诉求 |
| 极简占位页 | 仅显示头像和输入框 | 原型表现力弱 |
