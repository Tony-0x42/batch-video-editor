# 批量剪辑 部署说明

> 适用对象：运维 / 后端 / APP 打包人员
> 工程位置：后端 `server/`（RuoYi 3.9.2，Spring Boot 3 + JDK17），APP `android/`（Android，Gradle）
> 后端默认端口 `8080`，应用上下文 `/`（无前缀）

---

## 1. 环境依赖

| 依赖 | 版本要求 | 用途 | 备注 |
|------|----------|------|------|
| JDK | 17 | 后端编译/运行 | 本机位于 `D:\tools\jdk17`；Maven 包装脚本已自动指向 |
| Maven | 3.6+ | 后端构建 | 本机经 `D:\tools\bin\mvn.cmd` 调用 |
| Node.js / npm | 16+ | 管理后台 ruoyi-ui 构建 | 本机经 `D:\tools\bin\npm.cmd` 调用 |
| MySQL | 5.7+ / 8.x | 业务数据库 `batchvideo` | 见 `ruoyi-admin/src/main/resources/application-druid.yml` |
| Redis | 5+ | 缓存 / Token / 验证码 | 默认 `localhost:6379`，见 `application.yml` |
| FFmpeg | 4.x+ | AI 云创视频分割/批量合成 | 配置项 `batch.video.ffmpeg-path`，见 3.2 |
| Python | 3.8+ | AI 去水印链接解析（抖音/小红书） | 依赖 `requests`、`playwright`（需 `playwright install chromium`），见 3.3 |

## 2. 数据库初始化

### 2.1 建库

```sql
CREATE DATABASE batchvideo DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

数据库连接配置在 `ruoyi-admin/src/main/resources/application-druid.yml`（默认 `jdbc:mysql://localhost:3306/batchvideo`，账号/密码 `root/root`，**生产环境必须修改**）。

### 2.2 脚本执行顺序（`sql/` 目录）

全新部署按以下顺序执行：

| 顺序 | 脚本 | 说明 |
|------|------|------|
| 1 | `ry_20260417.sql` | RuoYi 基础表（sys_* 系统表） |
| 2 | `quartz.sql` | 定时任务表（QRTZ_*） |
| 3 | `batch_business.sql` | 批量剪辑业务表：客户、首页内容、公告、教程、文档、算力日志、生成记录、二维码统计、全局参数、APP 版本等（已含 `avatar_url` 列） |
| 4 | `batch_customer_add_password.sql` | `batch_customer` 增加 `password` 列（APP 登录密码，BCrypt） |
| 5 | `fix_computing_power_remain.sql` | `batch_customer` 增加 `computing_power_remain` 列（剩余算力） |
| 6 | `batch_content_app.sql` | APP 内容扩展表：品牌专区、联系方式 |
| 7 | `batch_watermark_parse.sql` | AI 去水印解析记录表 |
| 8 | `batch_ai_video.sql` | AI 云创视频组 / 分镜头表 |
| 9 | `batch_ai_video_group_add_sort_weight.sql` | 视频组排序权重字段（APP 拖拽排序） |
| 10 | `batch_ai_video_task.sql` | AI 视频真实合成管线改造：`batch_video_generate_log` 扩展任务字段（group_id/progress/result_url/error_msg/clip_seed/consume_value），status 语义变更为 0 处理中 / 1 成功 / 2 失败，并写入生成成本全局参数 `batch.ai.generateCost` |
| 11 | `batch_menu.sql` | 管理后台动态菜单（批量剪辑管理目录及各功能菜单/按钮权限） |
| 12 | `batch_menu_ai_video_log.sql` | 「生成记录」菜单（`batch:aivideo:list`，依赖第 11 步的目录 3000） |

可选脚本：

| 脚本 | 说明 |
|------|------|
| `batch_test_data.sql` | 开发/测试演示数据（需先执行 batch_business.sql 与 batch_content_app.sql），**生产环境不要执行** |
| `batch_setup.sql` | 幂等的 APP 演示数据初始化（含 password 列存在性判断），仅开发环境使用 |
| `batch_customer_add_avatar.sql` | **仅旧库升级用**：为已存在的 `batch_customer` 补 `avatar_url` 列；全新部署无需执行（batch_business.sql 已含） |

> 旧库升级注意：若库中存在 AI 视频旧数据（status 旧语义 0 成功 / 1 失败），执行 `batch_ai_video_task.sql` 前先按脚本头部注释刷数。

## 3. 后端配置与启动

### 3.1 基础配置（`ruoyi-admin/src/main/resources/application.yml`）

- `server.port = 8080`，`server.servlet.context-path = /`
- `ruoyi.profile = D:/ruoyi/uploadPath`（文件上传根目录，Linux 改为如 `/home/ruoyi/uploadPath`；AI 视频素材/产出存于其下 `video/upload`、`video/split`、`video/output`）
- `spring.data.redis.*`：Redis 地址/密码
- `token.secret`：**生产环境必须修改为随机长密钥**（默认 `abcdefghijklmnopqrstuvwxyz`）
- `spring.servlet.multipart.max-file-size = 10MB` / `max-request-size = 20MB`：视频素材上传若超限需调大

### 3.2 批量剪辑生产必配项（`batch.*`，代码中 `@Value` 读取，均有开发默认值）

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `batch.app.server-url` | `http://localhost:8080` | 服务对外访问地址，用于生成注册二维码内容（`QrCodeUtil`），**必须配为生产域名/公网地址** |
| `batch.app.download-url` | `https://batch-video-editor.example.com/download` | APP 下载页地址；扫码接口 `GET /batch/qrcode/scan` 计数后 302 跳转至此，**必须配置** |
| `batch.app.default-password` | `123456` | 后台新增客户时的初始密码（BCrypt 入库），生产建议改为强密码并通知用户首登修改 |
| `batch.video.ffmpeg-path` | `ffmpeg` | FFmpeg 可执行文件路径；未加入 PATH 时配置绝对路径（如 `D:/tools/ffmpeg/bin/ffmpeg.exe` 或 `/usr/local/bin/ffmpeg`）。启动时后端会自检，不可用则 AI 分割/生成全部失败 |

生产 `application.yml` 示例：

```yaml
batch:
  app:
    server-url: https://api.your-domain.com
    download-url: https://www.your-domain.com/download
    default-password: ChangeMe@2026
  video:
    ffmpeg-path: /usr/local/bin/ffmpeg
```

### 3.3 Python 解析服务（AI 去水印）

- 脚本：`scripts/video_parse.py`（抖音/小红书无水印解析，stdout 输出 JSON）。
- 安装依赖：
  ```bash
  pip install requests playwright
  playwright install chromium
  ```
- 后端通过 subprocess 调用，查找顺序：环境变量 `VIDEO_PARSE_SCRIPT` 指定路径 → `{user.dir}/scripts/video_parse.py` → `{user.dir}/../scripts/video_parse.py`。
- Python 解释器：JVM 系统属性 `batch.python.cmd` > 环境变量 `PYTHON_CMD` > 默认 `python`。服务器上建议显式设置，如：
  ```bash
  export PYTHON_CMD=/usr/bin/python3
  # 或启动参数 -Dbatch.python.cmd=/usr/bin/python3
  ```

### 3.4 构建与启动

```bash
cd server
mvn clean package -DskipTests        # 本机使用 D:\tools\bin\mvn.cmd
java -jar ruoyi-admin/target/ruoyi-admin.jar
```

启动后验证：访问 `http://<host>:8080/`，后台默认账号 `admin/admin123`。

## 5. APP 打包（android）

### 5.1 后端地址配置

`app/build.gradle` 通过 Gradle 属性注入 `BuildConfig.API_BASE_URL`：

| 属性 | 默认值 | 说明 |
|------|--------|------|
| `API_BASE_URL` | `http://192.168.1.5:8080/` | 通用 Base URL（gradle.properties 中配置，**勿提交生产地址到个人本地文件以外的公共配置**） |
| `API_BASE_URL_DEBUG` | 同上 | debug 包专用 |
| `API_BASE_URL_RELEASE` | 同上 | release 包专用，**打包上架前必须指向生产 HTTPS 地址** |

本地调试可在 `gradle.properties` 追加，例如：

```properties
API_BASE_URL=http://192.168.1.5:8080/
API_BASE_URL_RELEASE=https://api.your-domain.com/
```

> Android 模拟器访问本机后端使用 `http://10.0.2.2:8080/`。

### 5.2 打包命令

```bash
cd android
gradlew.bat assembleRelease        # 需要 JDK17（gradle.properties 已配置 org.gradle.java.home）
```

产物：`app/build/outputs/apk/release/app-release.apk`（当前 release 未开启混淆，`minifyEnabled false`）。

### 5.3 上线前注意：明文流量

`app/src/main/AndroidManifest.xml` 当前 `android:usesCleartextTraffic="true"`（仅用于 HTTP 联调）。后端切 HTTPS 后应：

1. 将 `API_BASE_URL_RELEASE` 配为 `https://` 地址；
2. 将 `usesCleartextTraffic` 改为 `false`，或配置 `networkSecurityConfig` 仅允许调试环境明文（代码中已留 TODO 注释）。
