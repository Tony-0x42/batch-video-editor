# 登录页

## 1. 页面概述

- **页面定位**：APP 启动后的身份验证入口，用于校验用户身份并进入 APP 主界面。
- **用户角色**：所有使用批量剪辑 AI APP 的用户（分公司、服务商、个人账号）。
- **入口来源**：APP 冷启动后默认进入；用户退出登录后再次进入。

## 2. 页面元素与功能点

### 2.1 品牌展示区

- **元素**：APP Logo、产品名称「批量剪辑」、Slogan（后台可配置）。
- **数据约束**：Logo 与 Slogan 由后台下发，前端只读展示；后台未配置时展示默认 Logo 与空 Slogan。

### 2.2 账号输入区

- **手机号输入框**：
  - 占位文案：请输入手机号
  - 输入限制：仅允许输入 11 位数字
  - 格式校验：前端校验手机号是否为合法大陆手机号
- **密码输入区**：
  - 占位文案：请输入密码
  - 输入限制：支持字母、数字、特殊字符，长度 6-20 位
  - 支持显示/隐藏密码切换图标

### 2.3 登录按钮

- **元素**：蓝色主按钮，文案「登录」。
- **状态**：
  - 未输入手机号或格式不正确时按钮置灰不可点击
  - 输入合法且验证码/密码已填写后按钮高亮可点击
- **点击后**：
  - 展示加载动画
  - 调用登录接口
  - 登录成功后缓存登录态，跳转 APP 首页
  - 登录失败时弹窗提示失败原因

### 2.4 其他操作

- **忘记密码**：跳转密码找回页（若当前策略支持）。
- **注册入口**：跳转 APP 注册页，用户可通过扫码获取上级手机号后输入手机号和密码完成注册。
- **用户协议 & 隐私政策**：登录页面底部展示协议勾选框与协议链接，用户需勾选同意后方可登录。

## 3. 交互流程

1. 用户打开 APP，进入登录页。
2. 页面自动拉取后台配置的品牌 Logo 与 Slogan。
3. 用户输入手机号，前端实时校验格式。
4. 用户输入密码，勾选用户协议。
5. 点击「登录」按钮，前端校验：
   - 手机号是否为空/格式错误
   - 密码是否为空/长度不足
   - 用户协议是否已勾选
6. 校验通过后调用登录接口，展示加载状态。
7. 登录成功：缓存 Token、用户信息、算力数据，跳转首页。
8. 登录失败：根据错误码弹窗提示，常见错误包括手机号未注册、验证码错误、密码错误、账号已注销等。

## 4. 数据字段与来源

| 字段 | 类型 | 来源 | 说明 |
|---|---|---|---|
| Logo | URL | 后台配置 | 登录页顶部品牌 Logo |
| Slogan | String | 后台配置 | 品牌宣传语 |
| 手机号 | String | 用户输入 | 11 位数字 |
| 密码 | String | 用户输入 | 6-20 位，加密后提交 |
| 用户协议/隐私政策 URL | URL | 后台配置 | 点击跳转 WebView/H5 查看 |
| Token | String | 登录接口返回 | 缓存后用于后续接口鉴权 |

## 5. 异常处理与边界规则

- **手机号格式错误**：输入框失去焦点或点击登录时提示「请输入正确的手机号」。
- **密码为空或长度不足**：点击登录时提示「请输入 6-20 位密码」。
- **未勾选协议**：点击登录时提示「请阅读并同意用户协议和隐私政策」。
- **登录接口失败**：弹窗提示具体错误信息，如「手机号或密码错误」「账号不存在」「账号已注销，请联系管理员」。
- **网络异常**：提示「网络连接失败，请检查网络后重试」。
- **后台无品牌配置**：展示默认占位 Logo，不报错崩溃。

## 6. UI 与视觉规范

- 页面背景：建议使用白色或浅灰色背景。
- 品牌 Logo 居中展示，尺寸由设计稿确定。
- 输入框统一样式：圆角边框、聚焦时蓝色边框、错误时红色边框。
- 主按钮：蓝色填充、圆角、全宽或固定宽度居中，文案白色。
- 底部协议文字：灰色小字，协议名称为蓝色可点击。
