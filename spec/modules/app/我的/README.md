# 我的（个人中心）

## 1. 页面概述

- **页面定位**：底部导航「我的」入口，展示用户账号信息、VIP 信息、账号服务、系统设置及退出登录入口。
- **用户角色**：所有已登录用户。
- **入口来源**：底部导航栏「我的」。
- **视觉规范**：重要字颜色为蓝色，和前面页面颜色统一。

## 2. 页面元素与功能点

### 2.1 账号基础信息区

- **元素**：头像、账号名称、脱敏手机号、VIP 标识、VIP 有效期。
- **展示字段**：
  - 脱敏手机号：示例「159*****1415」
  - VIP 信息：VIP 标识、VIP 有效期（格式 YYYY-MM-DD，例：2029-05-24）
- **编辑入口**：点击头像或「编辑账号资料」进入账号资料编辑页。
- **数据约束**：全部 VIP 时长、账号基础信息由后台下发管理。

### 2.2 账号服务功能列表

以列表形式展示以下功能入口：

| 入口 | 功能说明 |
|---|---|
| 联系客服 | 展示客服联系方式、服务时段（早 9:00-18:00），客服信息后台可编辑更新 |
| 抖音账号管理 | 抖音账号绑定、解绑、切换 |
| 绑定微信 | 微信授权绑定 / 解除绑定 |
| 切换账号 | 多账号登录切换 |
| 注销账号 | 账号注销流程（含二次确认、数据清除提示） |

### 2.3 系统 & 合规设置

| 入口 | 功能说明 |
|---|---|
| 用户协议 | 页面展示完整文档，文档内容后台可更新 |
| 隐私政策 | 页面展示完整文档，文档内容后台可更新 |
| 清理缓存 | 展示当前缓存占用大小（例：395.46MB），点击一键清理本地缓存 |
| 检查版本 | 检测 APP 最新版本，支持更新弹窗 |
| 版本信息展示 | 当前版本号、运行环境版本 |

### 2.4 底部操作

- **退出登录按钮**：蓝色按钮，点击后二次确认，确认后清除登录态并返回 APP 登录页。

## 3. 交互流程

1. 用户点击底部导航「我的」进入本页面。
2. 页面拉取用户基础信息、VIP 信息、客服信息、版本信息等。
3. 用户点击各功能入口进入对应子页面。
4. 用户点击「退出登录」，二次确认后退出并跳转登录页。

## 4. 数据字段与来源

| 字段 | 类型 | 来源 | 说明 |
|---|---|---|---|
| 头像 URL | URL | 后台/用户上传 | 用户头像 |
| 账号名称 | String | 后台/用户编辑 | 用户昵称/姓名 |
| 手机号 | String | 后台 | 脱敏展示 |
| VIP 标识 | Boolean | 后台 | 是否 VIP |
| VIP 有效期 | Date | 后台 | 格式 YYYY-MM-DD |
| 客服信息 | Object | 后台配置 | 电话、服务时段 |
| 缓存大小 | String | 本地计算 | 如「395.46MB」 |
| 当前版本号 | String | 本地/接口 | APP 当前版本 |
| 最新版本号 | String | 接口 | 用于检测更新 |

## 5. 异常处理与边界规则

- **后台信息拉取失败**：展示本地缓存数据或提示「信息加载失败，请下拉刷新」。
- **清理缓存失败**：提示「清理失败，请重试」。
- **版本检测失败**：提示「版本检测失败，请稍后重试」。
- **注销账号**：必须二次确认，并提示数据清除风险。
- **退出登录**：二次确认后清除本地登录态。

## 6. UI 与视觉规范

- 一级页面，底部有固定导航栏。
- 顶部账号信息区可设置蓝色渐变背景或白色卡片。
- VIP 标识、VIP 有效期等重要文字蓝色高亮。
- 功能列表项：左侧图标 + 中间标题 + 右侧箭头。
- 退出登录按钮为蓝色填充主按钮，位于页面底部。
