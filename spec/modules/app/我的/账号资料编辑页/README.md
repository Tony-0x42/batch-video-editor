# 账号资料编辑页

## 1. 页面概述

- **页面定位**：「我的」页面中编辑账号资料的二级页面，用于修改头像、昵称、手机号等基础信息。
- **用户角色**：所有已登录用户。
- **入口来源**：我的 → 账号基础信息区 / 编辑账号资料入口。

## 2. 页面元素与功能点

### 2.1 顶部导航栏

- **返回按钮**：点击返回「我的」页面。
- **页面标题**：「编辑资料」或「账号资料」。
- **保存按钮**：右上角「保存」按钮，点击后提交修改。

### 2.2 头像编辑

- **元素**：圆形头像 + 相机图标。
- **功能**：点击后唤起底部弹窗，选择「拍照」或「从相册选择」上传新头像。
- **限制**：图片大小、格式需符合后台要求。

### 2.3 表单字段

- **账号名称/昵称**：
  - 输入框，限制字符长度（如 2-20 个字符）
  - 必填
- **手机号**：
  - 展示当前手机号，支持点击更换手机号
  - 更换手机号需验证码校验
- **其他资料字段**：根据后台配置，可能包括姓名、性别、地区、简介等。

### 2.4 保存操作

- **元素**：底部蓝色「保存」按钮（或右上角保存按钮）。
- **状态**：未修改时置灰，修改后高亮可点击。
- **点击后**：校验表单，调用更新接口，成功后提示「保存成功」并返回。

## 3. 交互流程

1. 用户从「我的」页面进入账号资料编辑页。
2. 页面展示当前账号资料。
3. 用户修改头像、昵称或其他字段。
4. 用户点击「保存」，前端校验输入合法性。
5. 校验通过后调用更新接口，展示加载状态。
6. 更新成功后提示并返回「我的」页面，刷新账号信息。

## 4. 数据字段与来源

| 字段 | 类型 | 来源 | 说明 |
|---|---|---|---|
| 头像 | File/URL | 用户上传/后台 | 圆形头像 |
| 昵称 | String | 用户输入 | 2-20 字符 |
| 手机号 | String | 后台 | 可更换，需验证码 |
| 其他资料 | Object | 后台配置/用户输入 | 性别、地区、简介等 |

## 5. 异常处理与边界规则

- **昵称为空或超限**：提示「昵称长度需在 2-20 个字符之间」。
- **头像上传失败**：提示「头像上传失败，请重试」。
- **保存失败**：提示「保存失败，请检查网络后重试」。
- **手机号更换失败**：根据错误码提示验证码错误、手机号已存在等。

## 6. UI 与视觉规范

- 顶部返回按钮 + 标题 + 右上角保存按钮。
- 头像居中展示，下方可显示用户名。
- 表单项左侧标签，右侧输入框/箭头。
- 保存按钮蓝色填充，未修改时置灰。
