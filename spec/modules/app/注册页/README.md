# 注册页

## 1. 页面概述

- **页面定位**：新用户通过扫码或其他方式进入 APP 后，完成账号注册的页面。
- **用户角色**：未注册的新用户（潜在的分公司、服务商、个人用户）。
- **入口来源**：登录页 → 注册入口；微信扫码下载 APP 后首次启动进入。

## 2. 页面元素与功能点

### 2.1 品牌展示区

- **元素**：APP Logo、产品名称「批量剪辑」。
- **数据约束**：由后台下发，前端只读展示。

### 2.2 上级手机号展示

- **元素**：页面展示从二维码中读取到的上级手机号（脱敏展示）。
- **规则**：
  - 若用户通过微信扫码注册，自动读取二维码内上级手机号并展示。
  - 若上级手机号为空（无码注册），展示「暂无上级」或提示联系管理员挂靠。
  - 若二维码已失效（上级账号已注销），提示「渠道二维码已失效，请联系总部管理员重新获取」。

### 2.3 手机号输入区

- **元素**：手机号输入框。
- **占位文案**：请输入手机号
- **输入限制**：仅允许输入 11 位数字
- **格式校验**：前端校验手机号是否为合法大陆手机号
- **唯一性校验**：提交时后端校验手机号是否已被注册

### 2.4 密码输入区

- **元素**：密码输入框 + 确认密码输入框。
- **占位文案**：请输入密码 / 请再次输入密码
- **输入限制**：6-20 位，支持字母、数字、特殊字符
- **显示/隐藏切换**：支持切换密码可见性

### 2.5 用户协议 & 隐私政策

- **元素**：页面底部展示协议勾选框与协议链接。
- **规则**：用户必须勾选同意《用户协议》和《隐私政策》后方可注册。

### 2.6 注册按钮

- **元素**：蓝色主按钮，文案「注册」。
- **状态**：
  - 手机号、密码、确认密码、协议勾选未满足时置灰
  - 全部填写合法后高亮可点击
- **点击后**：
  - 前端校验手机号、密码一致性、协议勾选
  - 调用注册接口，提交手机号、密码、上级手机号
  - 注册成功后自动登录或跳转登录页
  - 注册失败时提示具体原因

### 2.7 返回登录入口

- **元素**：页面底部「已有账号？去登录」文字链接。
- **功能**：点击返回登录页。

## 3. 交互流程

1. 用户通过扫码下载并首次打开 APP，或从登录页点击注册入口进入注册页。
2. 页面读取二维码中的上级手机号参数并展示。
3. 用户输入手机号、设置密码、确认密码。
4. 用户勾选用户协议和隐私政策。
5. 点击「注册」，前端校验：
   - 手机号格式是否正确
   - 密码是否符合长度要求
   - 两次密码输入是否一致
   - 协议是否已勾选
6. 校验通过后调用注册接口。
7. 注册成功：自动登录并跳转首页，或跳转登录页提示注册成功。
8. 注册失败：根据错误码提示，如「手机号已注册」「二维码已失效」「上级手机号不存在」等。

## 4. 数据字段与来源

| 字段 | 类型 | 来源 | 说明 |
|---|---|---|---|
| 上级手机号 | String | 二维码参数 | 注册自动绑定层级的唯一凭证 |
| 手机号 | String | 用户输入 | 11 位数字，全局唯一 |
| 密码 | String | 用户输入 | 6-20 位，加密后提交 |
| 确认密码 | String | 用户输入 | 与密码一致 |
| 用户协议/隐私政策 URL | URL | 后台配置 | 点击跳转查看 |
| Token | String | 注册/登录接口返回 | 自动登录时返回 |

## 5. 异常处理与边界规则

- **手机号格式错误**：提示「请输入正确的手机号」。
- **手机号已注册**：提示「该手机号已注册，请直接登录」。
- **密码长度不足**：提示「密码长度需在 6-20 位之间」。
- **两次密码不一致**：提示「两次输入的密码不一致」。
- **未勾选协议**：提示「请阅读并同意用户协议和隐私政策」。
- **二维码已失效**：页面加载时提示「渠道二维码已失效，请联系总部管理员重新获取」，并禁用注册按钮或允许无上级注册。
- **上级手机号不存在/类型不匹配**：注册时后端校验，提示「上级账号不存在或类型不匹配」。
- **网络异常**：提示「网络连接失败，请检查网络后重试」。

## 6. UI 与视觉规范

- 页面背景白色或浅灰色。
- Logo 居中展示。
- 上级手机号展示为只读信息，可脱敏处理。
- 输入框圆角边框，聚焦时蓝色边框。
- 注册按钮蓝色填充、全宽或固定宽度居中。
- 底部「已有账号？去登录」蓝色可点击。
