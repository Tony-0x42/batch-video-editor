# 视频分割进度页

## 1. 页面概述

- **页面定位**：AI 视频分割后的独立进度展示页，实时展示视频切片处理进度。
- **用户角色**：所有已登录用户。
- **入口来源**：视频组编辑页 → 点击「AI 视频分割」并选择素材后进入。

## 2. 页面元素与功能点

### 2.1 顶部导航栏

- **返回按钮**：不建议退出（进度进行中可后台静默处理）。
- **页面标题**：「视频分割中」或「处理进度」。

### 2.2 进度展示

- **元素**：进度条/百分比数字 + 当前处理视频名称。
- **展示字段**：
  - 总视频数（如 5/10 个视频）
  - 当前视频切片进度（如 35%）
  - 预计剩余时间（可选）

### 2.3 切片配置信息

- **元素**：展示用户配置的切片时长。
- **展示字段**：切片时长（如 2.5s）。

### 2.4 后台静默处理提示

- **元素**：提示文案「切分完成后将自动返回编辑页」。
- **功能**：用户可将 APP 切换至后台，处理继续进行；完成后通过通知或再次进入 APP 时返回编辑页。

### 2.5 完成状态

- **元素**：处理完成后展示「完成」提示，自动跳转回视频组编辑页。
- **功能**：用户也可手动点击「完成」按钮返回。

## 3. 交互流程

1. 用户在视频组编辑页选择本地视频并配置切片时长。
2. 点击确认后进入视频分割进度页。
3. 页面调用分割接口，开始上传并处理视频。
4. 前端轮询或接收推送，实时更新进度。
5. 分割完成后，自动按原始上传顺序分组展示分镜头素材。
6. 页面自动返回视频组编辑页，用户继续后续操作。

## 4. 数据字段与来源

| 字段 | 类型 | 来源 | 说明 |
|---|---|---|---|
| 任务 ID | String | 接口返回 | 分割任务唯一标识 |
| 总视频数 | Number | 用户选择 | 单次最多 10 个 |
| 当前处理视频索引 | Number | 接口返回 | 如第 3 个 |
| 切片时长 | Number | 用户配置 | 0.5~10s，步长 0.1s |
| 当前进度百分比 | Number | 接口返回 | 0~100 |
| 任务状态 | Enum | 接口返回 | 待处理、处理中、已完成、失败 |

## 5. 异常处理与边界规则

- **分割失败**：提示「视频分割失败，请检查素材格式或网络后重试」，提供重试按钮。
- **网络中断**：提示「网络连接中断，恢复后自动继续」或「请检查网络后重试」。
- **用户退出页面**：后台继续处理，用户再次进入时恢复进度展示。
- **上传素材超限**：若超过 10 个视频，进入页面前提示「单次最多选择 10 个视频」。

## 6. UI 与视觉规范

- 进度条蓝色填充，底色浅灰。
- 进度百分比数字蓝色高亮。
- 页面中央展示大进度条，下方展示当前处理文件名。
- 完成时展示成功图标 +「处理完成」文案。
