# 新增账号页

## 1. 页面概述

- **页面定位**：客户管理模块下的二级页面，用于总后台管理员新增分公司、服务商或个人账号。
- **用户角色**：总后台管理员。
- **入口来源**：客户管理 → 新增账号按钮。

## 2. 页面元素与功能点

### 2.1 顶部导航栏

- **返回按钮**：点击返回客户管理列表页。
- **页面标题**：「新增账号」。

### 2.2 账号类型选择

- **元素**：单选或下拉选择。
- **选项**：分公司 / 服务商 / 个人。
- **交互**：选择不同类型后，下方表单项动态变化。

### 2.3 基础表单

- **账号名称**：
  - 输入框，必填
  - 限制长度（如 2-50 字符）
- **联系人**：
  - 输入框，必填
  - 限制长度（如 2-20 字符）
- **当前账号手机号**：
  - 输入框，必填
  - 前端校验 11 位手机号格式
  - 提交时后端校验全局唯一

### 2.4 上级手机号

- **分公司**：无上级，无需填写，表单中隐藏或展示「无上级」。
- **服务商**：必填所属分公司管理员手机号，系统校验该手机号为分公司类型账号。
- **个人**：必填所属服务商手机号，系统校验该手机号为服务商类型账号。

### 2.5 配额配置

- **新增分公司**：
  - 最大可创建服务商数量 X
  - 旗下服务商总可分配个人账号总量 Y
- **新增服务商**：
  - 可拆分创建个人账号上限 X
- **算力总配额 (GF)**：
  - 数字输入框，必填
  - 单位 GF
- **VIP 有效期**：
  - 日期选择器，必填
  - 格式 YYYY-MM-DD

### 2.6 提交操作

- **元素**：底部「保存」与「取消」按钮。
- **保存**：
  - 前端校验必填项与格式
  - 调用新增账号接口
  - 成功后自动生成微信可扫码的专属下载注册二维码
  - 成功后弹窗提示并可选择「查看二维码」或「返回列表」
- **取消**：返回客户管理列表页，不保存任何数据。

## 3. 交互流程

1. 管理员从客户管理列表点击「新增账号」。
2. 进入新增账号页，选择账号类型。
3. 根据类型填写基础信息、上级手机号、配额、算力、VIP 有效期。
4. 点击「保存」，系统进行前置校验：
   - 手机号是否重复
   - 上级手机号是否存在且类型匹配
   - 上级剩余名额是否充足
5. 校验通过后创建账号，系统自动生成注册二维码。
6. 提示创建成功，管理员可选择查看二维码或返回列表。

## 4. 数据字段与来源

| 字段 | 类型 | 来源 | 说明 |
|---|---|---|---|
| 账号类型 | Enum | 用户选择 | branch_company / service_provider / individual |
| 账号名称 | String | 用户输入 | 必填 |
| 联系人 | String | 用户输入 | 必填 |
| 当前账号手机号 | String | 用户输入 | 11 位，全局唯一 |
| 上级手机号 | String | 用户输入 | 服务商/个人必填 |
| 最大服务商数量 | Number | 用户输入 | 分公司类型必填 |
| 个人账号总容量 | Number | 用户输入 | 分公司类型必填 |
| 个人账号上限 | Number | 用户输入 | 服务商类型必填 |
| 算力总配额 | Number | 用户输入 | 单位 GF，必填 |
| VIP 有效期 | Date | 用户输入 | 必填 |
| 注册二维码 | URL | 系统自动生成 | 内置上级手机号参数 |

## 5. 异常处理与边界规则

- **手机号重复**：提示「该手机号已被注册，请更换手机号」。
- **手机号格式错误**：提示「请输入正确的手机号」。
- **上级手机号不存在**：提示「上级手机号不存在」。
- **上级手机号类型不匹配**：提示「上级手机号类型不匹配，请选择正确的上级」。
- **上级剩余名额不足**：提示「该上级可创建下级名额已满」。
- **必填项为空**：提示「请完善表单信息」。
- **保存失败**：提示「创建失败，请稍后重试」。

## 6. UI 与视觉规范

- 顶部返回按钮 + 标题。
- 表单分区块展示：账号类型、基础信息、上级绑定、配额配置、VIP 与算力。
- 输入框圆角边框，必填项红色星号标记。
- 日期选择器、数字输入框使用统一组件。
- 保存按钮蓝色填充，取消按钮灰色/白色边框。
