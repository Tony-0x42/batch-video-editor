# 注册二维码页

## 1. 页面概述

- **页面定位**：客户管理模块下的二级页面，用于查看、下载、复制、重置账号的专属注册二维码。
- **用户角色**：总后台管理员。
- **入口来源**：客户管理列表 → 二维码按钮；账号详情页 → 二维码区域。

## 2. 页面元素与功能点

### 2.1 顶部导航栏

- **返回按钮**：点击返回上一页。
- **页面标题**：「注册二维码」。

### 2.2 二维码展示

- **元素**：大尺寸的二维码图片。
- **信息**：
  - 二维码内置参数：绑定创建源头的上级手机号，作为注册自动绑定层级的唯一凭证。
  - 二维码图片可微信扫码识别。

### 2.3 账号信息展示

- **元素**：二维码下方展示该二维码所属账号信息。
- **字段**：
  - 账号名称
  - 账号类型
  - 上级手机号
  - 创建时间

### 2.4 操作按钮

- **下载二维码**：点击下载二维码图片到本地。
- **复制推广链接**：点击复制带参数的 APP 下载页面链接。
- **重置二维码**：二次确认后重置，旧二维码立即失效，系统生成新二维码。

### 2.5 数据统计

- **元素**：展示该二维码的推广数据。
- **字段**：
  - 扫码次数
  - APP 下载量
  - 成功注册用户数

## 3. 交互流程

1. 管理员进入注册二维码页。
2. 页面加载二维码图片与账号信息。
3. 管理员可下载二维码、复制推广链接。
4. 管理员点击「重置二维码」，二次确认后生成新二维码，旧二维码失效。
5. 用户微信扫码 → 跳转 APP 下载页面 → 安装后注册时自动读取二维码内上级手机号完成绑定。

## 4. 数据字段与来源

| 字段 | 类型 | 来源 | 说明 |
|---|---|---|---|
| 二维码图片 URL | URL | 后台生成 | 含上级手机号参数 |
| 推广链接 | URL | 后台生成 | 下载页面链接 + 上级手机号参数 |
| 账号名称 | String | 后台数据 | 二维码所属账号 |
| 账号类型 | Enum | 后台数据 | 分公司/服务商/个人 |
| 上级手机号 | String | 后台数据 | 绑定参数 |
| 扫码次数 | Number | 后台统计 | 二维码被扫描次数 |
| 下载量 | Number | 后台统计 | 通过二维码下载 APP 数量 |
| 注册用户数 | Number | 后台统计 | 通过二维码成功注册数量 |

## 5. 异常处理与边界规则

- **二维码生成失败**：提示「二维码生成失败，请重试」。
- **下载失败**：提示「下载失败，请检查浏览器权限」。
- **复制失败**：提示「复制失败，请手动复制」。
- **旧二维码失效**：用户扫码注册时，若上级账号已注销，提示「渠道二维码已失效，请联系总部管理员重新获取」。
- **无码注册兜底**：用户直接下载 APP 自主注册，无上级参数则无层级归属；仅总后台可手动修改账号上级手机号完成挂靠。

## 6. UI 与视觉规范

- 顶部返回按钮 + 标题。
- 二维码居中展示，尺寸较大（如 300x300px）。
- 二维码下方账号信息使用灰色小字。
- 操作按钮横向排列：下载、复制、重置。
- 重置按钮使用橙色/红色警示色，点击后二次确认。
- 数据统计卡片白色背景，数字蓝色高亮。
