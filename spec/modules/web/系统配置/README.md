# 系统配置

## 1. 页面概述

- **页面定位**：管理后台中用于配置系统级参数、版本管理、缓存策略、全局视觉等的模块。
- **用户角色**：总后台管理员。
- **入口来源**：后台首页 → 侧边栏「系统配置」。

## 2. 页面元素与功能点

### 2.1 顶部 Tab 切换

- **元素**：顶部 Tab，切换不同配置类型。
- **Tab 项**：
  - 品牌配置
  - 版本管理
  - 全局参数
  - 日志管理（可选）

### 2.2 品牌配置

- **字段**：
  - APP Logo
  - 后台 Logo
  - 产品名称
  - Slogan
  - 主色调（默认蓝色，可配置色值）
  - 登录页背景图（可选）
- **功能**：上传/替换 Logo、图片，修改文案，保存后实时生效。

### 2.3 版本管理

- **元素**：APP 版本列表。
- **字段**：
  - 版本号
  - 平台（Android/iOS）
  - 更新类型（强制更新/提示更新/静默更新）
  - 更新内容
  - 下载链接
  - 发布时间
  - 状态
- **操作**：新增版本、编辑、删除、上架/下架。

### 2.4 全局参数

- **字段**：
  - AI 云创单次选择视频上限（默认 10）
  - AI 分割切片时长区间（默认 0.5~10s）
  - 切片时长步长（默认 0.1s）
  - 算力不足提示文案
  - 链接解析失败提示文案
  - 空状态占位图
  - 客服服务时段
- **功能**：修改全局参数后，APP 端下次启动/刷新时生效。

### 2.5 日志管理（可选）

- **元素**：后台操作日志列表。
- **字段**：操作人、操作时间、操作模块、操作内容、IP 地址。
- **功能**：查询、筛选、导出日志。

## 3. 交互流程

1. 管理员进入系统配置页面。
2. 切换 Tab 选择要配置的类型。
3. 管理员修改品牌信息、版本信息、全局参数等。
4. 点击「保存」，系统校验并保存配置。
5. 配置生效：
   - 品牌配置：刷新后生效
   - 版本管理：APP 端检测版本时生效
   - 全局参数：APP 端启动/刷新时拉取

## 4. 数据字段与来源

| 字段 | 类型 | 来源 | 说明 |
|---|---|---|---|
| 品牌信息 | Object | 用户输入/上传 | Logo、名称、Slogan 等 |
| 版本列表 | Array<Object> | 后台数据 | 版本号、平台、更新类型等 |
| 全局参数 | Object | 用户输入 | 各类系统参数 |
| 操作日志 | Array<Object> | 后台记录 | 可选 |

## 5. 异常处理与边界规则

- **Logo 上传失败**：提示「上传失败，请检查文件格式和大小」。
- **必填项为空**：提示「请完善配置信息」。
- **版本号重复**：提示「该版本号已存在」。
- **参数格式错误**：如切片时长区间不合法，提示具体错误。
- **保存失败**：提示「保存失败，请稍后重试」。

## 6. UI 与视觉规范

- 顶部 Tab 选中项蓝色下划线。
- 品牌配置区使用表单布局，Logo 展示预览。
- 版本管理使用表格 + 新增/编辑弹窗。
- 全局参数使用表单布局，数字输入框带单位说明。
- 保存按钮蓝色填充，位于表单底部或右上角。
