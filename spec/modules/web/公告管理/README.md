# 公告管理

## 1. 页面概述

- **页面定位**：管理后台中用于发布、编辑、下架平台公告的模块。
- **用户角色**：总后台管理员。
- **入口来源**：后台首页 → 侧边栏「公告管理」。

## 2. 页面元素与功能点

### 2.1 顶部操作区

- **新增公告按钮**：蓝色主按钮，点击弹出新增公告弹窗/跳转新增页。
- **搜索框**：支持按公告标题搜索。
- **筛选条件**：
  - 状态：已发布 / 已下架 / 全部
  - 发布时间范围

### 2.2 公告列表

- **元素**：表格展示所有公告。
- **表格字段**：
  - 公告标题
  - 公告类型（通知/活动/重要更新）
  - 发布人
  - 发布时间
  - 状态（已发布/已下架）
  - 阅读量（可选）
  - 操作
- **操作按钮**：
  - 查看：预览公告内容
  - 编辑：修改公告内容
  - 上架/下架：切换公告状态
  - 删除：删除公告（二次确认）

### 2.3 新增/编辑公告

- **元素**：弹窗或独立页面。
- **字段**：
  - 公告标题（必填）
  - 公告类型（单选）
  - 公告内容（富文本编辑器，必填）
  - 封面图（可选）
  - 发布状态（立即发布/暂存）
  - 发布时间（可选，定时发布）

### 2.4 公告预览

- **元素**：弹窗展示公告在 APP 端的展示效果。
- **展示内容**：标题、发布时间、封面图、正文。

## 3. 交互流程

1. 管理员进入公告管理页面。
2. 页面加载公告列表。
3. 管理员新增公告，填写标题、类型、内容、封面图等。
4. 管理员可编辑、上架、下架、删除公告。
5. 已发布公告在 APP 端展示（展示位置以产品规划为准，如我的页面公告入口或首页弹窗）。

## 4. 数据字段与来源

| 字段 | 类型 | 来源 | 说明 |
|---|---|---|---|
| 公告标题 | String | 用户输入 | 必填 |
| 公告类型 | Enum | 用户选择 | 通知/活动/重要更新 |
| 公告内容 | HTML/String | 用户输入 | 富文本，必填 |
| 封面图 | URL | 用户上传 | 可选 |
| 发布状态 | Enum | 用户选择 | 已发布/已下架/暂存 |
| 发布时间 | Date | 系统自动/用户选择 | 定时发布时使用 |
| 阅读量 | Number | 后台统计 | 可选 |

## 5. 异常处理与边界规则

- **标题或内容为空**：提示「请填写公告标题和内容」。
- **发布失败**：提示「发布失败，请稍后重试」。
- **删除公告**：二次确认后删除。
- **下架公告**：已发布公告下架后 APP 端不再展示。

## 6. UI 与视觉规范

- 顶部返回/标题 + 新增按钮 + 筛选区。
- 公告列表表格展示，状态使用标签颜色区分（已发布绿色、已下架灰色）。
- 富文本编辑器使用统一组件，支持文字、图片、链接。
- 预览弹窗模拟 APP 端展示效果。
